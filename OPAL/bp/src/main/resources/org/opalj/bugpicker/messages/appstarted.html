Getting started

First, you need to supply the bug picker with the project that you want to analyze. To do that, either navigate in the menu to File > Open or press Ctrl+O (Cmd+O on MacOS).

Tip: the BugPicker remembers the last project you loaded, so to get started quickly, you can either navigate in the menu to File > Load last project or press Ctrl+L (Cmd+L on MacOS).

Jar files, libraries, and source code

When you specify a project, you can add three kinds of files to it:

  1. Project classes are the classes that the bug picker will analyze. They can be specified as single class files, bundled in jar files, or by selecting a directory contains them.
  2. Library classes are classes that your project classes depend on. They will not be analyzed. Library classes can only be specified through jar files.
  3. Source code allows you to browse the source code of your project in addition to the bytecode while you inspect the issues identified by the analysis. Source code can only be specified by directories containing the source files.
    Note: source files are expected to be located in a directory path corresponding to their package name. E.g. the class com.example.some.package.Foo should be located at ${sourceDirectory}/com/example/some/package/Foo.
