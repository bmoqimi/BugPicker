Browsing the report

In the area above, you will find the issues that the analysis has identified in your project.

You can click on any underlined element to take a closer look at the issue.

  • Clicking on a class's name (found in the first column) will show that class's bytecode in the bytecode tab. If you specified a source directory and the source for this class could be found, it will be loaded in the source tab.
  • Clicking on a method's name (found in the second column) will first load bytecode (and, if available) source code of the class surrounding that method as above. Then, it will scroll both tabs to the method in question.
  • Clicking on a PC will load the bytecode of the respective class in the bytecode tab and then scroll the view to this PC.
  • Clicking on a line number will load the source code (if available) of the respective class in the source tab and then scroll the view to that line. If no source could be found, it will instead load the bytecode and scroll to the PC that most closely corresponds to the line number.

Tip:You can view this message again by navigating in the menu to Help > Browse Help > Browsing the report.
